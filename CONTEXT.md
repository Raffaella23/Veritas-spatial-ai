# ARCHITECTURAL VISION

VERITAS is transitioning from a monolithic HTML application into a modular Spatial AI framework.

The project follows a strict separation of responsibilities:

Presentation Layer
↓
Passive Viewer (HTML / Unity / WebXR)
↓
Python Core Engine
↓
Simulation Data (JSON)

The Viewer must never contain business logic.

All intelligence resides inside the Python Core.

---

Responsibilities:

- AI

- Physics logic

# CORE DESIGN PRINCIPLES

## Core-First

All algorithms belong inside `/core`.

Never inside HTML.

Never inside Unity.

Never duplicated.

---

## Dumb Viewer

The Viewer only renders.

The Core makes every decision.

---

## Single Source of Truth

All simulation state originates exclusively from:

/data/simulation_config.json

No duplicated configuration is allowed.

---

## Behaviour Driven

Everything visible inside the Viewer must be generated by the Behaviour Engine.

---

# RULE OF THREE

Every Core feature must support all three reference domains:

- Gaming
- Museum
- Airport

If a feature cannot support all three domains, its design must be reconsidered.

---

# ACCESSIBILITY

Accessibility belongs to the Core.

Compliance modules (WCAG, XAUR and future standards) remain optional extensions.

Rendering engines must remain independent from accessibility reasoning.

---

# SYNTHETIC PLAYERS

Synthetic Players are autonomous AI agents.

Future responsibilities include:

- Decision making
- Path selection
- Environment perception
- Memory
- Cognitive workload
- Accessibility simulation
- Event interaction

Primary implementation:

/core/behaviour.py

---

# GLB ASSET PIPELINE

All 3D assets are loaded dynamically.

Each asset entry contains:

- URL
- Position
- Rotation
- Scale
- Metadata
- Accessibility tags
- Simulation properties

The Viewer must never hardcode assets.
